如题 这篇写一写servlet 问就是正好学到这

先套个公式 这是一个典型的web.xml文件配置

那么第一行<servlet-name></servlet-name> 非常简单啊就是给这个servlet起一个名字

<servlet-class></servlet-class>指定实现该 Servlet 功能的 Java 类的完全限定名（包括包名）。例如com.djtu.GetRequestDataServlet 是具体实现 Servlet 功能的类。

这两行就定义了一个servlet组件

下面的servlet-mapping元素用于将一个 URL 模式（或多个模式）映射到前面定义的 Servlet，使得当用户访问特定 URL 时，服务器知道让哪个 Servlet 处理请求。

<servlet-name></servlet-name>这中间的名字应与上面的servlet-name同名，表明这个映射是针对该 Servlet 的。

<url-pattern></url-pattern>用于定义一个或多个 URL 模式。在这个例子中，当用户访问 /getRequestData.do 这个路径时，请求将由 getRequestDataServlet 处理。而 /getRequestData.do 应位于资源文件中，在后台，它将会触发与这个路径匹配的 Servlet。
